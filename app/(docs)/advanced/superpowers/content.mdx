# Superpowers Plugin

This page is intentionally late in the site. Superpowers is most helpful after the team already understands [Daily Workflow](/workflows/daily) and [Project Setup](/configuration).

## Install or verify it through the plugin manager

Start here:

```text
/plugin
```

Look for `superpowers@claude-plugins-official`.

- If it is already installed, you are done.
- If it is missing and the repo genuinely benefits from it, install it from the plugin manager.

Then confirm the available skills:

```text
/skills
```

## What it changes

Superpowers adds a verification-heavy operating style around common workflows. In practice that usually means:

- stronger RED -> GREEN discipline
- more explicit evidence before Claude claims success
- more review pressure between implementation steps
- a richer skill surface for planning and execution

## Use it as reinforcement, not as training wheels

Superpowers works best when the team already understands the underlying workflow. If people still do not understand why verification comes first, a plugin will not solve that.

## Explicit commands

If you want to invoke it directly, use the names your environment exposes. Common examples include:

| Invoke | What it does |
| --- | --- |
| `/superpowers:brainstorm` | Requirements exploration |
| `/superpowers:write-plan` | Plan generation |
| `/superpowers:execute-plan` | Execution against the plan |
| `/using-superpowers` | Remind Claude about the active Superpowers surface |

## Practical advice

- check `/plugin` before assuming installation is required
- let the team learn the workflow first
- add Superpowers because it solves a real repo need, not because it sounds powerful
