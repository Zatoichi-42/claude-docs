## How to use this page

### If you are brand new

Read the official overview and quickstart, then come back to this site’s tutorials.

### If you are setting up a repo

Read settings, skills, and hooks in that order. That mirrors the gentlest path into shared configuration.

### If you lead a team

Track release notes regularly, keep one eye on community practice, and standardize only the patterns your team has already proven internally.
